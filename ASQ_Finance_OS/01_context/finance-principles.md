# Finance Principles

## 1. Capture first, clean later

A messy note is better than no record.

## 2. The system should reduce avoidance

Any finance workflow that requires too much manual maintenance will fail.

## 3. Every subscription needs a job

A tool should have a clear purpose, experiment, client use, or learning value.

## 4. Trials need cancel-by dates

Every trial should have a cancel-by date before the charge date.

## 5. Monthly burn informs pricing

ASQ Ashlee pricing should account for software costs, admin time, delivery time, taxes, and owner pay.

## 6. Tax certainty belongs to a CPA

This repo can create CPA-ready context, but it does not replace professional tax advice.

## 7. Records should be portable

Use plain text, Markdown, JSONL, and exportable formats whenever possible.
