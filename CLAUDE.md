This repository is organized into bounded operational workspaces.

Each workspace contains:

# Operational Rules

- Keep context bounded to the active workspace
- Do not blend unrelated client/project information
- Separate persistent intelligence from working artifacts
- Maintain human review between operational stages
- Prefer modular workflows over monolithic systems
- Structure before automation
- Pipelines before agents
