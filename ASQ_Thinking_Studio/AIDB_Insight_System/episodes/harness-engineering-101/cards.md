## Card 2 — HE101-007
**Format:** Team Prompt
**Audience:** Enterprise AI Champion, Company Champion
**Channel:** Slack, team meeting, internal newsletter

---

AIDB Team Prompt — Harness Engineering 101

Key idea:
The question most enterprise AI programs are asking is: which model should we use? That's the wrong question. Harness engineering reframes it — the model is necessary but insufficient. The environment you design around it determines whether it works.

Why it matters:
Your team's AI results aren't just a function of which tool you bought. They're a function of the context, workflows, and feedback loops you've built around it. That's something your team controls.

Team question:
If the model we're using stayed exactly the same for the next six months, what would we change about the environment we've built around it to get better results?

AIDB Reality Check — Harness Engineering 101

The moment:
In 2023 everyone talked about prompt engineering. In 2025 it was context engineering. Now there's a new term taking over — and if you've been using Claude Code, Cursor, or any AI agent, you've already been doing it without calling it that.

NLW's point:
Harness engineering is everything you put around the model — the systems, tooling, and access that help it do what it's meant to do. The frame has shifted from how you talk to the model, to what information it has, to what environment it works in. Each shift made the previous one feel smaller.
