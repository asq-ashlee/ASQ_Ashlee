# CLAUDE.md — AIDB Insight System

## What workspace this is

This is the AIDB Insight System workspace, part of ASQ Ashlee / ASQ Thinking Studio.

It is a working build — not a framework, not a deck, not a theory. It is a system being constructed in real time to test whether AIDB episodes can be decomposed into structured insight objects that travel further, land harder, and connect across time better than episode summaries alone.

## Where to find things

- `CONTEXT.md` — full thesis, thread structure, schema, and routing (read this second)
- `_config/` — brand constraints, audience definitions, voice guide (Layer 3, stable)
- `references/` — rubric, card templates, insight object schema, ICM principles (Layer 3, stable)
- `stages/` — four stage contracts with inputs, process, outputs (Layer 2)
- `episodes/` — working artifacts per episode (Layer 4, changes each run)

## How to navigate

If you are starting a new task, read this file first, then `CONTEXT.md`, then the relevant stage `CONTEXT.md`.

If you are working on a specific episode, go to `episodes/[episode-folder]/` for the working artifacts.

If you are updating reference material, go to `references/` or `_config/`.

## The one principle that governs everything

Portable does not mean contextless.
