# AIDB Growth Engineer — Experiment Design

This folder is the working system behind my application for the AIDB Growth Engineer role.

It is not a pitch deck. It is a designed experiment — ready to run in the first 30 days.

The core idea: most podcast growth treats the episode as the unit of distribution. This system tests the insight card as the unit instead. A short, recipient-specific package built from one high-signal moment. The episode doesn't need to travel first. The idea does.

## What's in here

- `rubric.md` — how to decide which moments become cards
- `card-formats.md` — the three card templates and when to use each
- `experiment.md` — the designed first experiment: hypothesis, method, signals, success criteria
- `example-cards.md` — two finished cards from "Who Cares About Consumer AI," ready to use as reference

## What this is not

This is not the full system. The full system — episode ingest, decomposition, audience routing, agent instructions, distribution studio — lives in the broader ASQ Ashlee architecture. That exists. This folder is the smallest useful version: one experiment, designed to run, with clear signals.

Scrappy version this week beats polished version next quarter.
